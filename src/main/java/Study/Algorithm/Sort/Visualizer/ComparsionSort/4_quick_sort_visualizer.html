퀵 정렬(양방향 포인터 방식, Hoare Parition) 시각화

피벗을 기준으로 분할·정복하며 한 번 확정된 피벗은 이후 연산에서 제외

📖 퀵 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 분할 정복(Divide And Conquer) — 문제를 작은 2개로 분리·해결
  • 하나의 값을 피벗(기준)으로 잡고, 왼쪽은 작은 배열·오른쪽은 큰 배열로 분할(Divide)
  • 이 과정이 Partition — 피벗 기준 재배치 & 분할
  • 재귀 호출마다 하나의 원소(Pivot) 위치 확정
  • 개선된 방법의 pivot 중간값은 이미 or 거의 정렬 or 역정렬 된 상황에서 매우 강함
  • 불안정 정렬(Unstable Sort), Merge와 달리 비균등 분할
  • 평균적으로 가장 빠른 정렬 & CPU Cache 를 잘 활용해 일반적인 정렬은 Quick 을 가장 먼저 고려
  • 주기억장치(RAM | ROM)에서 데이터를 가져올 때 필요한 데이터뿐만 아닌 주변 메모리(캐시 라인)까지 함께 Cache 에 저장
  • Quick Sort는 배열이 연속된 인덱스에 주로 접근해 CPU Cache 히트율이 높다
  • Heap Sort는 배열을 이진 트리처럼 사용해 부모(0) → 자식(1, 2) → 손자(3, 4, 5, 6) 처럼
  • 멀리 떨어진 인덱스를 반복 접근하기 떄문에 CPU Cache 히트율이 낮음
  • 따라서 같은 O(n log n)이라도 Quick Sort가 더 빠른 경우가 많아
  • CPU Cahce 히트율이 높은 Primitive 배열 (메모리상에 데이터를 연속으로 저장하는 타입들, char, int ...), 메모리 여유, 불안정 정렬도 괜찮은 경우엔 Quick 을 가장 먼저 고려

✅ 장점

  • 불필요한 데이터 이동을 줄이고 먼 거리 교환 가능
  • 한 번 확정된 피벗은 이후 연산에서 제외
  • O(n log n) 알고리즘 중 실제로 가장 빠름
  • 제자리 정렬 — 추가 메모리 공간 O(1)

❌ 단점 & 최악 케이스

  • 불안정 정렬
  • 피벗이 Min | Max이면 파티션이 안 나뉘어 O(n²)
  • 이미 오름차 | 내림차 정렬된 배열에서 최악
  • 정렬된 배열엔 Quick Sort의 불균형 분할로 오히려 느려짐
  • 시간 복잡도
    • Best O(n log n) Avg O(n log n) Worst O(n²)
  • 공간 복잡도
    • 추가 메모리 O(1) Avg 재귀 호출 비용 O(log n) Worst 재귀 호출 비용 O(n)

❤️ Lomuto & Hoare Parition

  • 구현한 코드 방식은 Hoare Parition (양방향 포인터), Lomuto Parition (단방향 포인터) 방식도 있음
  • Lomuto 방식은 pivot 이 오른쪽에 고정, 포인터인 i 는 partition 도중 pivot 보다 작은 값이 끝나는 인덱스 위치
  • 두 번째 포인터인 j 는 검사중인 인덱스 위치
  • i = 0 j = 0 부터 검사, j++ 와 pivot 을 비교 후 pivot 보다 작으면 i++ & swap(i, j), 아니면 j++ 로 계속 비교
  • j 가 pivot 앞까지 도달하면 swap(i, pivot) 하며 pivot은 위치 확정, 이후 재귀 호출로 분할 정복

⚡ 개선: medianOfThree 피벗

  • Random 피벗 — 최악을 확률적으로 회피, 단 Random.nextInt() 호출 비용 & 실행마다 성능 편차
  • 중간값 피벗(medianOfThree) — left·mid·right 중 중앙값 선택
  • Random보다 중간값이 항상 더 안정적으로 성능 개선
  • 완전히 O(n²)를 막을 순 없지만 O(n log n)에 가깝게 유지

quickSort() — 재귀 분할 
// left >= right 이면 원소 1개 이하 → 종료
private static void quickSort(int[] arr, int left, int right) {
  if (left >= right) { return; }

  // 개선된 피벗 선택 (medianOfThree)
  int pivotIndex = medianOfThree(arr, left, right);

  // 선택된 피벗을 첫 번째 위치로 이동
  swap(arr, left, pivotIndex);

  // 파티션 수행: 피벗 최종 위치 반환
  int pivot = partition(arr, left, right);

  // 피벗 기준 왼쪽·오른쪽 재귀
  quickSort(arr, left, pivot - 1);
  quickSort(arr, pivot + 1, right);
}
medianOfThree() — 피벗 선택 개선 
// 피벗이 Min|Max이면 파티션이 안 나뉘어 O(n²)
// → 오름차|내림차 정렬된 배열에서 최악
// 개선: left·mid·right 중 중앙값을 피벗으로 선택
// → 확률적으로 O(n log n) 유지, Random보다 안정적
private static int medianOfThree(int[] arr, int left, int right) {
  int mid = left + (right - left) / 2;

  // 기존 중앙값 탐색은 엣지케이스 발생 (예: 5, 5, 3)
  // → left·mid·right를 최소&중앙&최대로 먼저 정렬
  if (arr[left] > arr[mid])   { swap(arr, left, mid);   }
  if (arr[left] > arr[right]) { swap(arr, left, right); }
  if (arr[mid]  > arr[right]) { swap(arr, mid,  right); }

  // 이 시점에서 arr[mid]가 중간값
  return mid;
}
partition() — 피벗 기준 재배치 
// 피벗을 기준으로 왼쪽은 작은 값, 오른쪽은 큰 값으로 재배치
// 오른쪽에서 pivot보다 작은 값 찾고, 왼쪽에서 pivot보다 크거나 같은 값 찾아 스왑
// i, j 인덱스가 교차(더 바꿀 값 없음)하면 해당 인덱스와 피벗을 스왑 → 피벗 위치 확정
private static int partition(int[] arr, int left, int right) {
  int pivot = arr[left];
  int i = left, j = right;

  while (i < j) {
    // 오른쪽에서 pivot보다 작은 값 찾기
    while (i < j && arr[j] > pivot)    { j--; }
    // 왼쪽에서 pivot보다 크거나 같은 값 찾기
    while (i < j && arr[i] <= pivot)   { i++; }

    if (i < j) { swap(arr, i, j); }
  }

  // 피벗을 최종 위치(i)로 이동 → 이 위치는 이후 연산에서 제외
  swap(arr, left, i);
  return i;
}
