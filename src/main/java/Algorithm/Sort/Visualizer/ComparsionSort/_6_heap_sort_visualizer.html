힙 정렬 시각화

최대 힙(Max Heap) 구성 후, 루트를 추출하며 반복 정렬

📖 힙 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 완전 이진 트리 형태의 Heap 자료구조를 이용한 정렬
  • 최대 힙(Max Heap)을 구성해 루트(배열의 첫 번째 요소)에 최댓값을 위치
  • 루트와 마지막 원소를 Swap 후 힙 크기 1 감소
  • 힙 크기가 1이 될 때까지 Heapify & Swap 반복 → 오름차순 정렬
  • 부모 인덱스 i → 왼쪽 자식 2i+1, 오른쪽 자식 2i+2
  • 최댓값/최솟값을 여러 번 꺼내야 할 때 사용 (게임 랭킹, 우선순위 큐, 스케줄러)
  • 최악의 성능 보장이 필요할 때 적합 — 항상 O(n log n)

✅ 장점

  • Best/Avg/Worst 모두 O(n log n) 시간 보장
  • 제자리 정렬 — O(1) 추가 공간
  • 부분 정렬에 효율적 (가장 큰/작은 k개 원소 추출)
  • 우선순위 큐(Priority Queue) 원리 이해에 최적
  • 메모리 제약 환경에서 유용
Best O(n log n) Avg O(n log n) Worst O(n log n) Space O(1)

⚠️ 단점

  • 불안정 정렬 — 동일 값의 원래 순서 보장 안 됨
  • 구현이 복잡하고 상수 요소가 높음
  • 캐시 지역성 낮음 — 배열을 이진 트리처럼 사용해 멀리 떨어진 인덱스를 반복 접근
  • 부모(0) → 자식(1,2) → 손자(3,4,5,6) 처럼 불규칙한 메모리 접근 → CPU Cache 히트율 낮음
  • 같은 O(n log n)인 Quick Sort에 비해 실제로는 느린 경우가 많음
  • 입력 데이터 상태(정렬 여부)에 관계없이 항상 비슷한 수행 시간

🔎 Bottom-Up Heapify (Floyd), Top-Down Heapify 차이

  • Bottom-Up (Floyd 방식)
    • 이미 존재하는 배열을 heapify-down으로 정리, 전체 구조를 한 번에 구성, Build Heap 시간복잡도 O(n)
  • Top-Down (Insertion 방식)
    • 하나씩 삽입하며 sift-up으로 heap 유지, 실시간 데이터 삽입 구조, Build Heap 시간복잡도 O(n log n)
  • 차이 핵심
    • Top-Down: 삽입 기반 (Priority Queue, streaming system)
    • Bottom-Up: 일괄 구성 기반 (heap sort, batch processing)
    • 실제 사용:
    • Bottom-Up: 배열 -> Heap 변환, Heap Sort 구현
    • Top-Down: Priority Queue, 실시간 top-K, 스케줄링

heapSort() — 진입점 (Build 방식 선택: 1=Bottom-Up, 2=Top-Down) 
private static void heapSort(int[] arr, int selectNum) {
  int n = arr.length;

  // ① Build Max Heap — 방식 선택
  switch (selectNum) {
    case 1: // Bottom-Up (Floyd) → O(n)
      for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
      }
      break;
    case 2: // Top-Down (Insertion) → O(n log n)
      for (int i = 1; i < n; i++) {
        siftUp(arr, i);
      }
      break;
  }

  // ② Sort: 루트(최댓값)를 마지막 원소와 교환
  // 힙 크기 1 감소 → heapify 반복 → O(n log n)
  for (int i = n - 1; i > 0; i--) {
    // 루트 ↔ 마지막, 최댓값을 제자리로
    swap(arr, 0, i);
    // 축소된 힙에서 루트부터 재구성
    heapify(arr, i, 0);
  }
}

private static void swap(int[] arr, int a, int b) {
  int temp = arr[a];
  arr[a] = arr[b];
  arr[b] = temp;
}
siftUp() — Top-Down 삽입 방식 
// 새로 삽입된 노드를 부모와 비교하며
// 위쪽 방향으로 끌어올려 힙 속성 유지
// 시간복잡도: O(log n), 전체 Build: O(n log n)
private static void siftUp(
    int[] arr, int i) {

  int child = i;

  while (child > 0) {
    int parent = (child - 1) / 2;

    // 부모가 자식보다 크거나 같으면 종료
    if (arr[parent] >= arr[child]) { break; }

    // 부모 < 자식 → Swap 후 한 칸 위로 이동
    swap(arr, parent, child);
    child = parent;
  }
}
heapify() — 최대 힙 속성 재구성 
// 현재 노드를 기준으로 최대 힙 속성을
// 만족하도록 아래 방향으로 재구성
// 시간복잡도: O(log n)
private static void heapify(
    int[] arr, int n, int i) {

  int p = i;           // 최대값 후보 (처음엔 부모)
  int l = i * 2 + 1; // 왼쪽 자식 인덱스
  int r = i * 2 + 2; // 오른쪽 자식 인덱스

  // 왼쪽 자식이 부모보다 크면 후보 갱신
  if (l < n && arr[p] < arr[l]) { p = l; }

  // 오른쪽 자식이 현재 후보보다 크면 후보 갱신
  if (r < n && arr[p] < arr[r]) { p = r; }

  // 부모가 자식보다 작으면 Swap 후 재귀
  if (i != p) {
    swap(arr, p, i);
    // 교환된 위치에서 재귀
    heapify(arr, n, p);
  }
}
