계수 정렬(Counting Sort) 시각화

값의 등장 횟수를 세어(Counting) 비교 없이 정렬 — 안정 버전 vs 불안정 버전 비교

📖 계수 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • Non-Comparison Sort — 값 자체를 인덱스로 사용, 비교(Comparison) 없음
  • 비교 정렬의 O(n log n) 하한을 우회 가능
  • 정렬하려는 값이 특정 범위 내에 있을 때 사용
  • 값을 인덱스로 변환해 빈도를 누적하고 위치를 직접 계산하는 정렬
  • 등장 횟수를 세고(Counting) -> 누적합(Prefix Sum) -> 뒤에서부터 배치
  • countArr[i]는 값이 min + 1 이하인 원소의 개수
  • k(값의 범위)는 최댓값이 아니라 max - min + 1
  • 성능은 n이 아닌 k(값의 분포 범위)에 의해 결정

⚡ 안정 vs 불안정

  • Stable(countingSort): 누적합 + 뒤에서부터 배치 -> 동일 값 원소의 원래 순서 유지
  • Unstable(간단 구현): count 개수만큼 앞에서부터 값만 채워 넣음
  • Unstable 버전은 원본 위치 정보를 아예 저장하지 않음
  • 정수만 정렬할 땐 결과가 같아 보이지만, satellite data(딸린 필드)가 있으면 문제됨

✅ 장점

  • 시간 복잡도 O(n + k) — 비교 정렬보다 빠를 수 있음
  • Stable 버전은 안정 정렬 보장
  • 구현이 비교적 단순함
  • Radix Sort의 서브 루틴으로 활용 가능
Time O(n + k)

❌ 단점

  • k(범위)가 클수록 Counting 배열 크기가 커져 메모리 낭비
  • k가 n보다 훨씬 크면 비효율적 (ex: 범위가 1조 -> O(k))
  • 실수(float), 문자열 등 값 자체를 인덱스로 못 쓰는 데이터엔 부적합
  • 음수 포함 시 min 오프셋 계산이 반드시 필요
Space O(n + k)

🎯 어느 상황에 적용하면 좋은가

적합한 경우
  • 시험 점수(0~100점) 등 범위가 좁고 정해진 정수 정렬
  • 나이, 등급, 우편번호 앞자리처럼 값의 종류가 적은 데이터
  • 문자 빈도수 계산, 아나그램(Anagram) 판별
  • Radix Sort의 자릿수별 서브 루틴
  • 정렬뿐 아니라 도수분포표(히스토그램) 자체가 목적일 때
부적합한 경우
  • 주민등록번호, 큰 난수처럼 k(범위)가 n보다 훨씬 큰 데이터
  • 실수(float/double)처럼 값을 그대로 배열 인덱스로 못 쓰는 데이터
  • 문자열 정렬 (별도의 처리 없이는 인덱스화 불가)
  • satellite data(다른 필드가 딸린 객체)가 있는데 불안정 버전을 쓰는 경우

countingSort() vs unStableCountingSort() 차이
countingSort() — Stable
누적합으로 "값 이하 원소 개수"를 구한 뒤, 원본 배열을 뒤에서부터 순회하며 counting[v]-- 후 그 위치에 배치. 원본 원소 하나하나를 그대로 이동시키므로 동일 값 원소들의 상대적 순서가 보존됨.
unStableCountingSort() — Unstable
counting 배열에 등장 횟수만 기록해두고, 어떤 원소가 어디서 왔는지는 저장하지 않음. 인덱스 순서대로 "값 i가 counting[i]번 등장했다"는 사실만으로 새로 채워 넣기 때문에, 동일 값 원소들의 원래 순서를 재현할 방법이 없음 (위 시각화에서는 이해를 돕기 위해 역순으로 표현).
📄 소스 코드
countingSort() — Stable 
public static int[] countingSort(int[] arr) {
  if (arr == null || arr.length == 0) { return arr; }
  int min = arr[0], max = arr[0];

  for (int num : arr) {
    if (num < min) { min = num; }
    if (num > max) { max = num; }
  }

  int range = max - min + 1;
  int[] counting = new int[range];
  int[] output = new int[arr.length];

  for (int num : arr) { counting[num - min]++; }

  // 누적합: counting[i] -> "값 (i+min) 이하 원소 개수"
  for (int i = 1; i < range; i++) {
    counting[i] += counting[i - 1];
  }

  // 뒤에서부터 순회 + 위치 감소 배치 -> 안정 정렬 보장
  for (int i = arr.length - 1; i >= 0; i--) {
    int num = arr[i];
    counting[num - min]--;
    output[counting[num - min]] = num;
  }

  return output;
}
unStableCountingSort() — Unstable 
// counting[i]번 등장한 요소를 배열 순서 참조 없이 삽입 -> UnStable
public static int[] unStableCountingSort(int[] arr) {
  int min = arr[0], max = arr[0];
  for (int num : arr) {
    if (num < min) { min = num; }
    if (num > max) { max = num; }
  }

  int range = max - min + 1;
  int[] counting = new int[range];

  for (int num : arr) { counting[num - min]++; }

  int[] output = new int[arr.length];
  int idx = 0;
  // 누적합 없이, 등장 횟수만큼 값을 그대로 채워 넣음
  // -> 어떤 원소가 어디서 왔는지 정보가 없음
  for (int i = 0; i < range; i++) {
    while (counting[i] > 0) {
      output[idx++] = i + min;
      counting[i]--;
    }
  }
  return output;
}
