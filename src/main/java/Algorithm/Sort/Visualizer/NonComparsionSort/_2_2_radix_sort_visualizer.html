기수 정렬(Radix Sort) 시각화

데이터를 구성하는 자릿수(Radix)로 Counting Sort를 반복 — LSD(하위 자리부터) vs MSD(상위 자리부터) 비교

📖 기수 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 데이터를 구성하는 기본 요소(Radix, 자릿수)를 이용해 정렬
  • 입력값의 최대값이 클 때 Counting Sort의 비효율을 개선하기 위해 사용
  • 자릿수 값 별로 정렬하므로 각 자리에서 나올 수 있는 값의 범위는 0~9로 고정
  • 내부적으로 Counting Sort를 자릿수 단위로 반복 호출하는 구조
  • 안정 정렬(Stable), 자릿수가 없는 실수(부동소수점)는 정렬 불가
Time O(d · (n + b)) Space O(n + b)

⚡ LSD vs MSD — 정렬 방향

  • LSD(Least Significant Digit): 낮은(작은) 자리수부터 정렬
  • MSD(Most Significant Digit): 높은(큰) 자리수부터 정렬
  • LSD는 1600000과 1처럼 자릿수 차이가 크면 그 자릿수만큼 모두 비교해야 하는 단점
  • MSD는 마지막 자리수까지 확인할 필요 없이 상위 자릿수만으로 조기 종료 가능
  • LSD는 중간 정렬 결과를 알 수 없지만(10004 vs 70002), MSD는 중간에 상위 자릿수 순서를 바로 알 수 있음

✅ LSD 장단점

  • 장점
    • 구현 단순, 분기 없는 일관된 흐름(Branch-free)
    • 항상 배열을 순차 접근 → 캐시 지역성이 좋음
    • 데이터 분포와 무관하게 성능이 일정(예측 가능)
  • 단점
    • 자릿수(d)만큼 무조건 전부 순회, 상위 자릿수로 갈렸어도 활용 못 함
    • 고정 길이 키에 적합, 가변 길이 키(문자열)엔 패딩 등 추가 처리 필요

✅ MSD 장단점

  • 장점
    • 상위 자릿수만으로 그룹이 좁혀지면 조기 종료 가능
    • 가변 길이 키(문자열, prefix 비교)에 자연스럽게 적용, Top-K / 사전식 검색에 유리
  • 단점
    • 재귀 호출과 그룹별 분기로 흐름이 일관되지 않음
    • 데이터 분포에 따라 성능 편차가 큼(최악의 경우 LSD와 비슷하거나 더 느림)
    • 재귀 스택 공간 추가 필요, 경계 배열 관리 등 구현이 상대적으로 복잡

🎯 어느 상황에 적용하면 좋은가

LSD가 유리한 경우
  • 정수처럼 자릿수(길이)가 고정/균일한 데이터
  • 성능 예측 가능성이 중요한 실시간/하드웨어성 구현
  • 구현 단순성이 중요할 때, Branch-free 알고리즘이 필요할 때
MSD가 유리한 경우
  • 문자열처럼 키 길이가 가변적인 데이터
  • 상위 자릿수만으로 대부분 결정되는 랜덤 정수/텍스트
  • 전체 정렬을 끝까지 하지 않고 상위 기준 순서/일부 결과만 필요할 때

lsdRadixSort() vs msdRadixSort() 차이
LSD — 낮은 자리부터
1의 자리 → 10의 자리 → 100의 자리 순으로 배열 전체에 대해 Counting Sort를 반복 적용. 각 라운드는 이전 라운드의 정렬 결과 위에서 안정적으로 동작하며, 자릿수(exponent)가 최댓값을 넘어서면 종료. 재귀 없이 반복문(while)만으로 동작.
MSD — 높은 자리부터
가장 큰 자릿수를 기준으로 구간을 10개의 버킷(0~9)으로 나눈 뒤, 각 버킷(경계 배열 기준)에 대해 한 자리씩 낮춰가며 재귀 호출. 구간의 크기가 1 이하가 되거나 더 볼 자릿수가 없으면 재귀 종료. 위 시각화에서 흐리게 표시된 막대는 현재 재귀에서 다루지 않는(다른 구간) 원소.
📄 소스 코드 — 공통 로직 먼저, 이후 LSD → MSD 순
공통 로직 — countSort() & negativeValueCorrection() (LSD/MSD가 함께 사용) 
// 특정 자릿수를 기준으로 counting sort 수행
// output, countArr 은 호출 전 1회 선언 한 배열을 계속 재사용
private static int[] countSort(int[] arr, int start, int end, long exponent, int[] countArr, int[] output) {
  int range = end - start;

  // 이전 호출에서 남은 값 초기화
  Arrays.fill(countArr, 0, 10, 0);

  // 빈도 계산, exp = 자릿수
  for (int i = start; i < end; i++) { countArr[(int) (arr[i] / exponent) % 10]++; }

  // 누적 합
  for (int i = 1; i < 10; i++) { countArr[i] += countArr[i - 1]; }

  // MSD의 재귀호출 에서 사용할 누적합 백업
  int[] boundaryArr = countArr.clone();

  // 안정 정렬
  for (int i = end - 1; i >= start; i--) {
    int digits = (int) (arr[i] / exponent) % 10;
    output[--countArr[digits]] = arr[i];
  }

  // 원래 배열에 정렬된 결과 복사
  System.arraycopy(output, 0, arr, start, range);

  return boundaryArr;
}

static int negativeValueCorrection(int[] arr, int min, boolean isRollback, int offset) {
  if (isRollback) {
    // 음수 보정 롤백
    if (min < 0) { for (int i = 0; i < arr.length; i++) { arr[i] -= offset; } }
  } else {
    // 음수 보정 (전체를 양수로)
    if (min < 0) {
      offset = -min;
      for (int i = 0; i < arr.length; i++) { arr[i] += offset; }
    }
  }
  return offset;
}
lsdRadixSort() — LSD 
// LSD(가장 낮은 수 부터 정렬)
private static void lsdRadixSort(int[] arr, int n) {
  if (arr.length == 0) {return;}
  int max = arr[0], min = arr[0];

  for (int num : arr) {
    max = Math.max(max, num);
    min = Math.min(min, num);
  }

  int offset = negativeValueCorrection(arr, min, false, 0);

  int[] output = new int[n];
  int[] countArr = new int[10];

  // 최댓값을 나눴을 때 0이 나오면 모든 숫자가 i의 아래
  for (long exponent = 1; (max / exponent) > 0; exponent *= 10) {countSort(arr, 0, n, exponent, countArr, output);}
  negativeValueCorrection(arr, min, true, offset);
}
msdRadixSort() & runMSDRadixSort() — MSD 
static void msdRadixSort(int[] arr, int n) {
  if (arr.length == 0) {return;}

  // 음수 보정 offset
  int min = arr[0];
  for (int num : arr) {min = Math.min(min, num);}
  int offset = negativeValueCorrection(arr, min, false, 0);

  int[] output = new int[n];
  int[] countArr = new int[10];

  runMSDRadixSort(arr, 0, n, getMaxDigits(arr), countArr, output);

  negativeValueCorrection(arr, min, true, offset);
}

// MSD
// 큰 자릿수 부터 Counting Sort 하여 누적합 counting이 되는 범위를 새 그룹의 범위로 설정 & 재귀호출
static void runMSDRadixSort(int[] arr, int start, int end, int maxDigits, int[] countArr, int[] output) {
  int range = end - start;
  if (range <= 1 || maxDigits < 0) {return;}

  // 자릿수 계산
  long exponent = (long) Math.pow(10, maxDigits);

  // 재귀호출 실행 전 사용 할 누적합 백업
  int[] boundaryArr = countSort(arr, start, end, exponent, countArr, output);

  // 각 버킷마다 재귀호출 실행
  int nextStart = start;
  for (int i = 0; i < 10; i++) {
    int nextEnd = start + boundaryArr[i];

    if (nextEnd > nextStart) {runMSDRadixSort(arr, nextStart, nextEnd, maxDigits - 1, countArr, output);}
    nextStart = nextEnd;
  }
}

private static int getMaxDigits(int[] arr) {
  // 자릿수 계산
  int max = 0;
  for (int v : arr) {max = Math.max(max, v);}

  int digits = 0;
  while (max > 0) {
    max /= 10;
    digits++;
  }

  return digits - 1;
}
