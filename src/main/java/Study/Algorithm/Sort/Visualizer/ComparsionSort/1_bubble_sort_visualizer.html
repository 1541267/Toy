버블 정렬 시각화

인접한 두 원소를 비교해 1회전마다 최댓값을 맨 뒤에 위치

📖 버블 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 서로 인접한 원소를 대소 비교, 교환 반복
  • 1회전 시 가장 큰 값이 맨 뒤로 이동
  • 최댓값 순서대로 뒤에서부터 확정
  • 안정 정렬(Stable Sort)
  • 실무에서 거의 사용 X

✅ 장점

  • 구현이 간단
  • 제자리 정렬 — 추가 메모리 공간 필요 없음
  • 안정 정렬

❌ 단점

  • O(n²) — 매우 비효율적
  • 정렬되지 않은 원소를 옮기기 위해 교환(Swap)이 많이 발생
  • 배열이 길수록 비효율적

⏱ 복잡도

  • Best (정렬 돼있어 swap이 되지 않은 경우) O(n), Avg & Worst O(n²)
  • 공간 복잡도: 배열 안에서 교환 → O(1)
Best O(n) Avg O(n²) Worst O(n²) Space O(1)

// 거품 정렬: 서로 인접한 원소를 대소비교, 1회전 시 가장 큰 값이 맨 뒤, 최대값 순서대로 정렬
// 시간복잡도: Best (정렬 돼있어 swap이 되지 않은 경우) O(n), Worst & Avg = O(n²)
// 공간복잡도: 주어진 배열 안에서 교환(Swap) O(1)
// 장점: 구현 간단, 제자리 정렬이라 다른 메모리 공간 필요 X, 안정 정렬(Stable Sort)
// 단점: O(n²) 비효율, 정렬되어 있지 않은 원소를 정렬하기 위해 교환 연산(Swap)이 많이 발생
// 배열의 길이가 길수록 비효율적

for (int i = 0; i < arr.length - 1; i++) {
  boolean swaaped = false;
  // j가 arr.length - i 까지만 순회 → 이미 확정된 뒤쪽은 제외
  for (int j = 1; j < arr.length - i; j++) {

    // 인접한 두 원소 비교, 앞이 크면 교환 → 큰 값이 오른쪽으로 밀림
    if (arr[j - 1] > arr[j]) {
      int temp = arr[j - 1];
      arr[j - 1] = arr[j];
      arr[j] = temp;
      swaaped = true;
    }
  }
  // 정렬돼있어 한번도 swap이 일어나지 않는 경우 조기 종료로 O(n)
  if (!swapped) { break; }
  // 1회전 완료 → 인덱스 (arr.length - 1 - i) 에 최댓값 확정
}
