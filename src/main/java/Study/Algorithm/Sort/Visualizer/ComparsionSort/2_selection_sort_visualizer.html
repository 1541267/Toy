선택 정렬 시각화

채울 위치(인덱스)를 먼저 선택하고, 남은 범위에서 최솟값을 찾아 배치

📖 선택 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 원소를 넣을 위치(인덱스)가 먼저 고정 — Bubble Sort와의 차이
  • 1회전 시 최솟값이 맨 앞으로, 최솟값 순서로 정렬
  • 시험에서 n회 회전 후 배열 모양 구하기 문제에 자주 등장
  • 불안정 정렬(Unstable Sort)
  • Swap 비용이 매우 큰 경우(Bubble Sort), Selction은 최대 n-1번
  • ex) 디스크, SSD ... 처럼 쓰기 횟수를 줄이고 싶은 경우

⚡ Selection vs Insertion

  • 둘 다 k번째 반복 이후 첫 k개 요소가 정렬된 상태
  • Selection: K+1 요소 찾기 위해 모든 요소 탐색
  • Insertion: K+1 요소 배치에 필요한 만큼만 탐색 → 더 효율적

✅ 장점

  • 알고리즘 단순
  • Bubble에 비해 비교 횟수가 적음
  • 교환이 많은 자료에서 비교적 효율적
  • 제자리 정렬 — 추가 메모리 공간 필요 없음

❌ 단점 / 복잡도

  • O(n²) — 비효율적
  • 불안정 정렬
  • 배열이 길수록 비효율적
Best O(n²) Avg O(n²) Worst O(n²) Space O(1)

// 선택 정렬: 원소를 넣을 위치(인덱스)는 정해져 있는 것이 Bubble Sort 와의 차이
// 1회전 시 최솟값이 맨 앞, 최솟값 순서로 정렬
// 시간 복잡도: Best & Worst & Avg = O(n²)
// 공간 복잡도: 주어진 배열 안에서 교환(Swap) O(1)
// 장점: 알고리즘 단순, Bubble에 비해 비교횟수가 적어 교환이 많은 자료에서 비교적 효율적
//       제자리 정렬이라 다른 메모리 공간 필요 X
// 단점: O(n²) 비효율, 불안정 정렬(Unstable Sort), 배열 길수록 비효율

int index, temp;

for (int i = 0; i < arr.length - 1; i++) {

  // i 번째 자리에 들어올 최솟값의 인덱스를 찾는 과정 시작
  index = i;

  for (int j = i + 1; j < arr.length; j++) {
    // 현재 최솟값보다 더 작은 값 발견 시 인덱스 갱신
    if (arr[index] > arr[j]) { index = j; }
  }

  // i 번째 원소와 찾아낸 최솟값 위치(index)를 교환
  temp = arr[index];
  arr[index] = arr[i];
  arr[i] = temp;
  // 1회전 완료 → 인덱스 i 에 최솟값 확정
}
