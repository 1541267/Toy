버킷 정렬(Bucket Sort) 시각화

데이터를 n개의 버킷으로 분산 → 각 버킷 내부를 Insertion Sort로 정렬 → 순서대로 합침 — 정수 vs 실수(0~1) 비교

📖 버킷 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 데이터를 여러 개의 버킷(bucket)으로 분산한 뒤 각 버킷 내부에서 정렬을 수행하고, 버킷을 순서대로 합쳐 완성
  • 핵심은 값을 index로 변환(mapping)하는 것 — 값의 상대적 위치를 기준으로 버킷에 배치
  • bucketIdx Integer Ver - (num - min) / (max - min) × (n - 1)
    • 어떤 값이든 0~n-1 범위로 압축(정규화)
  • bucketIdx Double Ver - (int) ((long) (num - min) * (n - 1) / max - min + 1)
    • (num - min) / (max - min) : 전체 범위에서 몇 % 위치인지, (n - 1) : bucket 개수만큼 늘림
  • 버킷 내부 데이터는 매우 적다는 가정 하에 보통 Insertion Sort를 사용 (캐시 친화적·구현 단순)
Best/Avg O(n + k) Worst O(n²)

🪣 버킷 자료구조 선택

  • List<List<>>: index 기반 접근으로 O(1), cache locality가 좋아 성능 우수 (기본 선택)
  • Map<Integer, List<T>>: 동적 버킷 생성 가능하지만 HashMap은 순서 보장 X + 해시 오버헤드 존재
  • TreeMap: key 순서는 보장되지만 O(log n) 오버헤드 추가
  • 분산 → 정렬 → 수집, 3단계로 이뤄짐 (각 버킷은 균등 크기로 n개 생성)

⚡ O(n)에 가까워지는 조건

  • 데이터가 균등 분포(uniform distribution)일 때
  • 각 버킷에 들어가는 원소 수가 O(1)에 가까울 때
  • 버킷 내부 정렬 비용이 작을 때 (Insertion Sort 적합)
  • Worst: 모든 데이터가 하나의 버킷에 몰리는 경우 → 내부 정렬이 O(n²)이면 일반 정렬보다 느려질 수 있음

🎯 실수 버킷의 부동소수점 오차

  • 실수 정렬에서 부동소수점 오차로 인해 bucketIdx == n이 나올 수 있음
  • 따라서 bucketIdx = Math.min(bucketIdx, n - 1)로 클램핑(clamp) 조정 필요
  • 정수 버전은 정수 나눗셈 특성상 이 문제가 발생하지 않아 클램핑이 불필요
  • 범위 정규화: 서로 다른 범위의 데이터를 동일한 버킷 구조에 넣기 위한 과정

🎯 정수 vs 실수 — 어느 쪽에 잘 맞는가

정수(Integer)에서 사용 시
  • 정수도 사용 가능하지만 범위가 너무 크면 버킷 크기 설정에 문제 발생 가능
  • 예: 1~1,000,000처럼 입력 범위가 너무 크면 메모리 낭비
  • 부동소수점 오차가 없어 클램핑(min(idx, n-1)) 불필요
실수(Double, 0~1)에서 사용 시
  • 연속적인 값이 일정 범위 내에 균등 분포할 때 가장 잘 작동
  • 0~1 범위처럼 O(n)에 가까운 성능 가능, 버킷당 원소 수가 적어짐
  • 부동소수점 오차 보정을 위한 클램핑이 반드시 필요

bucketSortIntegerVer() vs bucketSortDoubleVer() 차이
정수(Integer) 버전
bucketIdx = (long)(num - min) × (n - 1) / range 로 계산 (range = max - min + 1). 정수 나눗셈은 항상 0~n-1 범위 안에 정확히 떨어지므로 클램핑이 필요 없음. 대신 데이터 범위가 매우 크면(예: 1~1,000,000) 버킷 크기 설정에 문제가 생기고 메모리가 낭비될 수 있음.
실수(Double) 버전
bucketIdx = (num - min) / (max - min) × (n - 1) 로 정규화 후 부동소수점 오차 때문에 bucketIdx가 n이 되어버릴 수 있어 Math.min(bucketIdx, n - 1) 클램핑이 필수. 0~1처럼 균등하게 분포된 실수에서 O(n)에 가장 근접한 성능을 냄.
📄 소스 코드 — 공통 Insertion Sort 먼저, 이후 정수 → 실수 순
공통 로직 — insertionSortInteger() & insertionSortDouble() (버킷 내부 정렬) 
// 버킷 내부는 원소가 매우 적다는 가정 하에 Insertion Sort 사용
// cache 친화적이고 구현이 단순하며, 적은 데이터에서는 quick/merge sort보다 빠름
static void insertionSortInteger(List<Integer> list) {
  int n = list.size();

  for (int i = 1; i < n; i++) {
    int temp = list.get(i);
    int prev = i - 1;

    while (prev >= 0 && list.get(prev) > temp) {
      list.set(prev + 1, list.get(prev));
      prev--;
    }

    list.set(prev + 1, temp);
  }
}

// double 버전도 비교 연산자만 동일하게 재사용, 로직은 완전히 같음
static void insertionSortDouble(List<Double> list) {
  int n = list.size();

  for (int i = 1; i < n; i++) {
    double temp = list.get(i);
    int prev = i - 1;

    while (prev >= 0 && list.get(prev) > temp) {
      list.set(prev + 1, list.get(prev));
      prev--;
    }

    list.set(prev + 1, temp);
  }
}
bucketSortIntegerVer() — 정수 
private static void bucketSortIntegerVer(int[] arr) {
  int n = arr.length;
  if (n <= 1) {return;}

  int min = arr[0], max = arr[0];
  for (int val : arr) {
    if (val < min) {min = val;}
    if (val > max) {max = val;}
  }
  if (max == min) {return;}

  int range = max - min + 1;

  // List<List<>>가 기본, index 접근 O(1) & cache 친화적
  List<List<Integer>> buckets = new ArrayList<>();
  for (int i = 0; i < n; i++) {buckets.add(new ArrayList<>());}

  // 분산 단계: bucketIdx = (num - min) * (n - 1) / range
  for (int num : arr) {
    int bucketIdx = (int) ((long) (num - min) * (n - 1) / range);
    buckets.get(bucketIdx).add(num);
  }

  // 정렬 단계: 각 버킷 내부는 Insertion Sort
  for (List<Integer> bucket : buckets) {insertionSortInteger(bucket);}

  // 수집 단계: 버킷을 순서대로 합침
  int idx = 0;
  for (List<Integer> bucket : buckets) {
    for (Integer num : bucket) {arr[idx++] = num;}
  }
}
bucketSortDoubleVer() — 실수 
static void bucketSortDoubleVer(double[] arr) {
  int n = arr.length;
  if (n <= 1) {return;}

  // 0 ~ 1 범위 위한 정규화
  double min = arr[0], max = arr[0];
  for (int i = 1; i < n; i++) {
    max = Math.max(max, arr[i]);
    min = Math.min(min, arr[i]);
  }
  if (max == min) {return;}

  List<List<Double>> buckets = new ArrayList<>();
  for (int i = 0; i < n; i++) {buckets.add(new ArrayList<>());}

  // 분산 단계: 범위 정규화로 이뤄진 버킷 인덱스로 분산
  for (double num : arr) {
    int bucketIdx = (int) ((num - min) / (max - min) * (n - 1));
    // 부동소수점 오차 보정: bucketIdx == n 가능성 차단
    bucketIdx = Math.min(bucketIdx, n - 1);
    buckets.get(bucketIdx).add(num);
  }

  for (List<Double> bucket : buckets) {insertionSortDouble(bucket);}

  int idx = 0;
  for (List<Double> bucket : buckets) {
    for (Double num : bucket) {arr[idx++] = num;}
  }
}
