병합 정렬 시각화

요소를 최소 길이로 쪼갠 후 정렬, 이 후 병합 시키면서 정렬

📖 병합 정렬 학습 노트 (클릭해서 펼치기)

🔍 개념

  • 분할 정복 방식으로 구현
  • 빠른 정렬로 분류, Quick Sort와 함께 자주 언급
  • Quick과 반대로 안정 정렬(Stable Sort)
  • 요소를 최소 단위로 쪼갠 후, 병합하면서 정렬
  • 두 영역이 이미 정렬돼 있으므로 순차 비교로 병합
  • 데이터들의 크기가 주기억장치(RAM)에 다 들어가지 못하는 데이터 정렬에도 많이 사용
  • 안정 정렬이 필요한 경우 사용 ex) 동접자는 이름순 유지 ...

⚡ Quick vs Merge

  • Quick: 피벗으로 재배치 & 분할 (Partition)
  • Merge: 단순히 반으로 나눈 뒤 Merge 과정에서 정렬
  • LinkedList 정렬 → Merge Sort 유리
  • Quick Sort는 Partition 시 임의 접근·swap 필요 → LinkedList 비효율
  • 배열 접근 O(1) vs LinkedList 접근 O(n)

✅ 장점

  • 안정 정렬 (같은 값의 순서 보장)
  • 최악의 경우에도 O(n log n) 보장
  • LinkedList 정렬에 매우 효율적
  • 순차적 비교 → 캐시 예측 가능한 접근
Best O(n log n) Avg O(n log n) Worst O(n log n)

❌ 단점

  • 추가 메모리 필요 — 제자리 정렬 아님
  • 배열 복사 과정으로 메모리 효율 떨어짐
  • 작은 배열에선 삽입 정렬보다 느릴 수 있음
  • 캐시 친화적이지 않음 (메모리 접근 많음)
Space O(n)
merge() vs optimizedMerge() 차이
merge(): 매 병합마다 Arrays.copyOfRange() 로 L, R 두 배열을 새로 생성 → 메모리 할당 + GC 반복으로 무거움
optimizedMerge(): temp 배열을 최초 한 번만 생성 후 해당 구간만 복사해서 사용 → 왼쪽 잔여 원소만 복사하면 되고 오른쪽 잔여 원소는 arr에 그대로 남아 별도 복사 불필요

📄 소스 코드
mergeSort() — 공통 (분할 재귀) 
private static void mergeSort(
    int[] arr, int[] temp, int selectNum,
    int left, int right) {

  // left >= right 이면 원소가 1개 이하 → 더 이상 분할 불필요, 재귀 종료
  if (left < right) {

    // 중간 인덱스 계산 (left + right) / 2 대신 오버플로 방지를 위해 아래 방식 사용
    int mid = left + (right - left) / 2;

    // 왼쪽 절반 재귀 분할
    mergeSort(arr, temp, selectNum, left, mid);
    // 오른쪽 절반 재귀 분할
    mergeSort(arr, temp, selectNum, mid + 1, right);

    // 두 절반이 각각 정렬된 상태에서 병합
    switch (selectNum) {
      case 1:
        // 공부용: 매 merge 마다 L, R 배열을 새로 생성
        merge(arr, left, mid, right);
        break;
      case 2:
        // 개선: 미리 생성한 temp 배열의 구간만 복사해서 사용
        optimizedMerge(arr, temp, left, mid, right);
        break;
      default:
        throw new IllegalArgumentException("잘못된 정렬 선택");
    }
  }
}
merge() — 기본 (공부용) 
// merge 구간을 임시 두 배열로 복사
private static void merge(int[] arr, int left, int mid, int right) {

  int[] L = Arrays.copyOfRange(arr, left, mid + 1);
  int[] R = Arrays.copyOfRange(arr, mid + 1, right + 1);

  int i = 0, j = 0, k = left;
  int leftSize  = L.length;
  int rightSize = R.length;

  while (i < leftSize && j < rightSize) {
    // L[i] < R[j] 로 하면 오른쪽 값이 먼저 들어가
    // 안정 정렬이 깨짐 → <= 로 비교해야 안정 정렬 보장
    if (L[i] <= R[j]) {
      arr[k++] = L[i++];
    } else {
      arr[k++] = R[j++];
    }
  }

  // 왼쪽 잔여 원소 복사
  while (i < leftSize) {
    arr[k++] = L[i++];
  }

  // 오른쪽 잔여 원소 복사
  while (j < rightSize) {
    arr[k++] = R[j++];
  }
}
optimizedMerge() — 개선 (copyOfRange 최적화) 
// 기존: 매 merge 마다 Arrays.copyOfRange() 로 L, R 배열을 새로 생성
// → 메모리 할당 & GC 제거가 반복돼 무거움
// 개선: temp 배열은 최초 한 번만 생성
//       merge 시 필요한 구간만 temp 로 복사 후
//       temp 에서 읽고 arr 에 기록
private static void optimizedMerge(
    int[] arr, int[] temp,
    int left, int mid, int right) {

  // left ~ right 구간만 temp 에 복사
  System.arraycopy(arr, left, temp, left, right - left + 1);

  int i = left;      // 왼쪽 시작 포인터
  int j = mid + 1;  // 오른쪽 시작 포인터
  int k = left;      // arr 에 저장할 위치

  while (i <= mid && j <= right) {
    // <= 로 비교해야 안정 정렬 보장
    if (temp[i] <= temp[j]) {
      arr[k++] = temp[i++];
    } else {
      arr[k++] = temp[j++];
    }
  }

  // 오른쪽 원소가 먼저 선택되면 왼쪽 원소가 arr 에서 덮어써져 사라질 수 있음
  // → 남은 왼쪽 원소(temp[i ~ mid])는 반드시 다시 arr 로 복사
  //
  // 반면 오른쪽 잔여(temp[j ~ right])는
  // k 가 j 위치에 도달하기 전엔 arr 에 원래 값이 그대로 남아 있음
  // → 별도 복사 불필요, 왼쪽만 처리하면 됨
  while (i <= mid) {
    arr[k++] = temp[i++];
  }
}
