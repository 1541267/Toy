🔍 개념

  • Select, Bubble과 유사하나 동작 방식이 다름
  • 2번째 요소부터 시작해 앞의 요소들과 비교 후 삽입 위치 지정
  • 앞의 원소를 뒤로 옮기고 지정된 자리에 삽입
  • 안정 정렬(Stable Sort)
  • 삽입 정렬은 단독으로는 잘 안 쓰지만, 다른 정렬 알고리즘 속에서 보조 역할로 많이 사용
  • ex) 배열길이가 길 때 퀵 정렬로 재귀를 하다보면 작은 배열이 많이 생기는데, 이 경우 퀵 정렬를 계속 하는 것 보다 삽입 정렬이 더 빠름
    • Tim Sort(Merge + Insetion), Intro Sort(Quick + Heap + Insertion)

✅ 장점

  • 알고리즘 단순
  • 이미 정렬된 경우 O(n) — Best Case 매우 효과적
  • 제자리 정렬 — 추가 메모리 공간 불필요
  • 안정 정렬
  • Selection·Bubble 동일 O(n²) 중 상대적으로 빠름

❌ 단점

  • Worst & Avg = O(n²)
  • 역순 정렬 시 최악 — 매 삽입마다 전체 이동 발생
  • 배열 길이가 길수록 비효율적
Best O(n) Avg O(n²) Worst O(n²) Space O(1)

⚡ Selection vs Insertion

  • k번 반복 후 첫 k개가 정렬됨 — 둘 다 유사
  • Selection: K+1 요소를 위해 항상 나머지 전체 탐색
  • Insertion: K+1 요소 배치에 필요한 만큼만 탐색 — 더 효율적
  • 이미 정렬된 구간이 많을수록 Insertion이 압도적으로 유리

// Select, Bubble과 유사, 2번째 요소부터 시작
// 앞의 요소들과 비교 후 삽입 할 위치 지정, 앞의 원소를 뒤로 옮기고 지정된 자리에 삽입
// 시간복잡도: Best(정렬된 경우) O(n), Worst(역순 정렬) & Avg O(n²)
// 공간복잡도: 주어진 배열 안에서 Swap O(1)
// 장점: 단순, 정렬된 경우 매우 효과적, 추가 메모리 필요X, 안정 정렬
//       Select|Bubble과 같은 O(n²)보다 상대적으로 빠름
// 단점: Worst & Avg = O(n²), 배열 길이 길수록 비효율
// 팁: Selection, Insertion 모두 k번 반복 후 첫 k개가 정렬된 순서
//  → Selection: K+1 요소를 위해 항상 모든 요소 탐색
//  → Insertion: K+1 요소 배치에 필요한 만큼만 탐색 (더 효율적)

for (int i = 1; i < arr.length; i++) {
  int temp = arr[i];  // 삽입할 키 값 저장
  int prev = i - 1;  // 비교 대상 인덱스 (왼쪽 정렬 영역의 끝)

  // 키보다 큰 원소를 오른쪽으로 한 칸씩 이동
  while (prev >= 0 && arr[prev] > temp) {
    arr[prev + 1] = arr[prev];
    prev--;
  }

  // 찾은 위치(prev+1)에 키 삽입
  arr[prev + 1] = temp;
}
